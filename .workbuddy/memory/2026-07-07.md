# 2026-07-07 工作日志

## 今日工作

1. 讨论 TIM3 双中断（period-elapsed / OC）的任务分配架构。
   - 关键事实：TIM3 的 Update 和 CC1 中断同属一个 `TIM3_IRQHandler` 向量，由 HAL_TIM_IRQHandler 按标志顺序分发，二者不并行、不构成真正 pipeline。
   - 结论：分两个回调或一个回调性能等价，仅为代码组织清晰。
2. 用户追问"为何 PWM 下发要 16k / OC、compute→apply 顺序"。澄清：
   - 占空比只在速度环(4k)改变，PWM 下发写 CCR 在 16k = 每周期 3 次 no-op（last-value 检查后跳过），并非硬需求。
   - "compute→apply 顺序"在窄义上成立（OC 在 CNT=ARR/2 触发，晚于 period-elapsed 的 CNT=0，同一周期内先算后发），但同一 ISR 内等价。
   - 真正把 apply 放 16k/OC 的理由：① OC 是天然的"每 tick 心跳"，统一给 4 路推 CCR；② 能及时捕获速度环之外来源(Vofa/CLI/DroneCAN)的直接占空比指令，延迟≤62.5µs；③ 自愈任何异常 CCR 状态。
   - 电机 PWM 在 TIM1/8 上且 CCR 预装载，写入相位对真实输出无影响（下次 UEV 才生效），故 TIM3 的 OC 相位无关 PWM 正确性。
   - 等价替代：在 period-elapsed 速度环后立即 apply(4k) 也合法，速度环变更延迟更低，但会漏掉速度环之间的外部指令直到下个速度 tick。

## 编码器 SPI 性能 + HAL_ERROR 诊断（KTH7111 ×4 / SPI3 共享总线）

- **架构关键**：4 个 KTH7111 共用 SPI3，靠各自 CS 选通（非独立 SPI），每个编码器是独立芯片。故 `KTH7111_Init()` 必须每个实例各跑一次——不能"只跑一次"，否则 M2~M4 不会被配置（滤波/方向）。
- **`BurnMTP()` 不该每次上电跑**：它烧录非易失 MTP 且含 `vTaskDelay(500)`，4 轴 = 2s + 大量 SPI 折腾。应挪到工厂一次性校准函数；boot Init 只做轻量 live 配置（方向寄存器 + 滤波寄存器 0x0d，不烧录）。
- **SPI 配置（CubeMX）**：Half-Duplex Master / 8bit / Motorola / prescaler /16(10.5MHz)。Half-Duplex 必须手动切 BIDIOE（`SPI_Set_MOSI_Input/Output`），这是正确的，不能去掉。
- **`HAL_SPI_ERROR_FLAG` 根因**：共享 `hspi3` 句柄 + 4× `BurnMTP` 的 SPI 折腾，残留 OVR/MODF 错误标志未清除，下一次 `HAL_SPI_Transmit` 经 `SPI_CheckFlagError` 检测到即报此错（具体位在 `hspi->ErrorCode`：OVR/MODF/FRE/CRCERR）。与"是否接实物"无关（主模式发数据不需要从机应答）。
- **修复清单**：① `KTH7111_Init` 去掉 `BurnMTP`；② `KTH7111_Send_JIAO_CMD` 开头加 `__HAL_SPI_CLEAR_ERRFLAG(_motor_config->hspi)`；③ 确认 `spi.c` 与 `.ioc` 一致(8bit/Half-Duplex)；④ `NSS_PULSE` 改 `DISABLE` 更稳。
- **`My_Tim_Callback` 性能 bug**：每 160 tick 读 4 编码器 = 1584µs（~396µs/电机），远超 62.5µs 中断周期，饿死其他中断。修复：轮询 1 电机/tick（`count & 0x03`），每电机 4kHz；并修 `count%160`(=100Hz) 误为 4kHz 的 cadence bug（注释写 4k 但实际 100Hz）。

## ST-Link 连接 WSL 环境坑（已验证）

- **ST-Link 只在电脑某个特定 USB 口能枚举**，其他口识别不到（报"未知 USB 设备 描述符请求失败"）。固定插该口（疑似 USB2.0 直连主板，供电/枚举最稳）。
- 识别后 BUSID 形如 `4-4`（VID:PID `0483:3752`），STATE 初始 `Shared`（已 bind）。
- 挂 WSL：Windows 管理员 `usbipd attach --wsl --busid 4-4`，目标发行版 `dev`。attach 后 STATE 应变 `Attached`，WSL 内 `lsusb` 应见 `0483:3752`。
- **坑**：若 Windows 有程序占用 COM5（串口助手 / CubeProgrammer / IDE），attach 不会真正把设备交给 WSL（`lsusb` 看不到）。需先关占用程序再 attach。
- WSL 内 OpenOCD 已装（`/usr/bin/openocd`），`openocd.cfg` 正确（stlink + stm32g4x）。烧录：`sudo openocd -f openocd.cfg -c "program build/debug/122_Rocket_Projectile.elf verify reset exit"`。
